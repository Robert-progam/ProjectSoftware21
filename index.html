Function 9: Surveys display and Employee's Survey
Description:
This function prints all the surveys that have been made for the employees in the hotel. As it is not a manager task, he/she doesn't change, edit, or add any new survey, it is a display only, with the posibility to look for more information on each employee, by clicking on the name. SurveyData.java Creates a class that selects all the information about the surveys from a the Surveys table of the database. It also creates a class that selects all the surveys with a specific ID, so that the surveys of one employee can be seen.

Function 2: Display Event Hall Reservations
Description:
This function prints all the reservations that have been made. The manager can see the more information about the reservation by clicking on moreinfo, he can also erase the reservations that had been canceled and also add new reservations.
Use:
The servlet calls one of the classes created in EventHallData, that selects (through sql) all the reservations that have been made. By clicking on the link, it brings the manager to another servlet, called EventHallReservationInfo, where the manager can see the information of the client that has done that reservation. By clickin on the Erase link, it calls another servlet called EraseEventHall, that delets from the database the reservation selected
Finally, the manager can add a new reservation by clicking the new reservation button that calls another servlet called NewReservationHall, that with the help of some Ajax and another servelt, AddHallReservation, add a new reservation to the database.

It uses: "EventHallResv", "EventHallReservationInfo", "EraseEventHall", "AddHallReservation", "NewReservationHall" and some Ajax "checkreservationHall", "CheckDayHall".
EventHallReserv.java Prints all the reservation that have been made. It includes two links, the first one takes you to another servlet called EventHallReservationInfo and the second link, uses the EraseEventHall servlet to delet a reservation. There is also a new reservation button.
EventHallReservationInfo.java This servlet prints the information of the client that has done the reservation.
EraseEventHall.java Gets the ReservationID from the reservation that the manager wants to delete, and calls the EraseEventHall servlet to delete that reservation from the database
NewReservationHall.java This Servlet, displays a form, where the manager can add a new reservation. While the manager adds the information, there is another code that is running at the same time checking if the information is okey.
AddHallReservation.java This Servlet, gets the information from the NewReservationHall form, and adds that information on the database.
checkreservationHall.js This is a .js file. It is in charge to display a message if the day that is being introduced on the NewReservationHall already exists. If that occurs, it displays an error message. If it does not exist, it displays and okey message.
CheckDayHall.java This file is in charge to check if the day introduced on the NewReservationHall exists. This servlet looks for a reservation with the same day that the one introduced by the manager, if it exist, returns a vector with that information.
